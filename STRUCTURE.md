# 📁 وثيقة هيكل ملفات مشروع ثروة كابيتال
> **تاريخ الإصدار:** يونيو 2025 | **الإصدار:** 1.0
> وثيقة مرجعية شاملة لهيكل الملفات ووظيفة كل مكوّن في الموقع والإدارة ولوحة العميل

---

## 📐 نظرة عامة على البنية

```
src/
├── main.tsx                    ← نقطة الدخول الرئيسية للتطبيق
├── router.tsx                  ← إعداد TanStack Router
├── styles.css                  ← المتغيرات العامة وأنماط Tailwind
├── routeTree.gen.ts            ← ملف مولَّد تلقائياً (لا تعدّله)
│
├── routes/                     ← صفحات الموقع (كل ملف = مسار URL)
├── components/
│   ├── site/                   ← مكونات الموقع العام
│   ├── admin/                  ← مكونات لوحة الإدارة
│   └── ui/                     ← مكتبة shadcn/ui (مكونات جاهزة)
├── contexts/                   ← سياقات React (اللغة)
├── hooks/                      ← هوكات مخصصة
└── lib/                        ← مكتبات وأدوات مشتركة
```

## 1️⃣ الملفات الجذرية (src/)

| الملف | الوظيفة |
|-------|---------|
| `main.tsx` | نقطة الدخول — يُنشئ React root ويُشغّل RouterProvider |
| `router.tsx` | يُهيئ TanStack Router مع QueryClient (إدارة البيانات) وxscrollRestoration |
| `styles.css` | متغيرات CSS للألوان والخطوط (Cairo)، أنماط Tailwind العامة، تأثير الوميض |
| `routeTree.gen.ts` | **مولَّد تلقائياً** — شجرة المسارات، لا تعدّله يدوياً |

## 2️⃣ الموقع العام — `src/routes/`

> كل ملف `.tsx` في هذا المجلد يمثل صفحة مستقلة بمسار URL خاص بها.

### صفحات الموقع العام

| الملف | المسار | الوظيفة |
|-------|--------|---------|
| `__root.tsx` | `/` (جذر) | الغلاف الرئيسي — يضم Header، Footer، WhatsappButton، BackToTop، CookieBanner، ToastProvider، LanguageProvider. كل الصفحات تمر عبره |
| `index.tsx` | `/` | الصفحة الرئيسية — تجمع: Hero + LiveTicker + شارات الثقة + ServicesSection + StatsSection + MarketsPreview + LatestNews + CTA banner |
| `about.tsx` | `/about` | صفحة "من نحن" — التاريخ الزمني للشركة، الفريق القيادي، الأرقام والإنجازات (ثنائية اللغة عربي/إنجليزي) |
| `services.tsx` | `/services` | قائمة الخدمات الستة (أسهم خليجية، عالمية، رقمية، صناديق، معادن، طاقة) مع مقارنة العوائد والميزات |
| `service.$id.tsx` | `/service/:id` | صفحة تفصيلية لكل خدمة — يُعرض بناءً على المعرّف `id` (gulf-stocks, crypto...) مع الميزات، المخاطر، الحد الأدنى، FAQ خاص |
| `markets.tsx` | `/markets` | عرض الأسواق المالية — أسهم، رقمية، معادن، فوركس، نفط — مع أسعار حية محاكاة وفلتر بحث |
| `news.tsx` | `/news` | قائمة المقالات والتحليلات المالية مع فلتر التصنيف والبحث |
| `article.$slug.tsx` | `/article/:slug` | صفحة المقال المفصّل — يُعرض بناءً على `slug`، يحتوي مقدمة وجسم المقال والكاتب والتاريخ |
| `contact.tsx` | `/contact` | **صفحة التواصل** — نموذج (الاسم، البريد، الهاتف، الخدمة، الرسالة) مع التحقق. **عند الإرسال يُحفظ تلقائياً في localStorage ويظهر فوراً في لوحة الأدمن** |
| `faq.tsx` | `/faq` | الأسئلة الشائعة — 12+ سؤال منظّم بحسب التصنيف مع accordion وفلتر بحث |
| `login.tsx` | `/login` | صفحة دخول العملاء — تحقق من 6 حسابات تجريبية محددة مسبقاً، عند النجاح يُحفظ `clientAuth` في localStorage وينتقل للوحة العميل |
| `dashboard.tsx` | `/dashboard` | **لوحة تحكم العميل** — 7 تبويبات كاملة (تفصيل أدناه) |
| `Akadmin.tsx` | `/Akadmin` | **مدخل لوحة الإدارة** — يعرض `AdminLogin` ثم `AdminLayout` بعد التحقق |
| `sitemap[.]xml.ts` | `/sitemap.xml` | يُولّد خريطة الموقع XML لمحركات البحث |

## 3️⃣ لوحة تحكم العميل — `src/routes/dashboard.tsx`

> ملف واحد يحتوي 7 تبويبات كاملة للعميل المسجّل.

| التبويب | المفتاح | المحتوى |
|---------|---------|---------|
| 🏠 المحفظة | `portfolio` | ملخص المحفظة، توزيع الأصول بالرسم البياني، قائمة الأصول مع الأداء، ملف العميل الشخصي |
| 📰 التحديثات | `news` | الأخبار والتحليلات الخاصة بالعميل + إعلانات الأدمن |
| 📈 الأداء | `performance` | رسم بياني لأداء المحفظة عبر الزمن، مؤشرات (Sharpe، Volatility، Max Drawdown) |
| 📋 المعاملات | `transactions` | سجل تاريخي كامل للصفقات (شراء/بيع/تحويل) مع تصدير PDF |
| 💬 الدعم | `support` | **مرتبط بلوحة الأدمن** — نموذج إرسال رسالة للمستشار. **تُحفظ الرسالة في localStorage وتظهر في صفحة Messages بلوحة الأدمن تحت تصنيف "دعم العملاء"** |
| ⚙️ الإعدادات | `settings` | بيانات الحساب (بريد، كلمة مرور)، إعدادات الإشعارات |

**ربط بالبيانات:**
- يقرأ `clientAuth` من `localStorage` للتعرف على هوية العميل
- يقرأ إعلانات الأدمن (`adminAnnouncements`) من `localStorage`
- يكتب رسائل الدعم عبر `addContactMessage()` في store.ts

## 9️⃣ تدفق البيانات الكامل

### رسالة تواصل جديدة:
```
المستخدم يملأ /contact  →  addContactMessage()  →  localStorage
                                                        ↓
                                              dispatchEvent('tharwah_contact_messages_changed')
                                                        ↓
                                              Messages.tsx (تحديث فوري)
                                              AdminLayout.tsx (badge يتحدث)
```

### تعديل الأدمن لبيانات عميل:
```
Clients.tsx يُعدّل → setLiveClients() → localStorage
                                              ↓
                                     dashboard.tsx (يقرأ عند التحميل)
```

### تسجيل دخول العميل:
```
login.tsx → يتحقق من CLIENT_ACCOUNTS → يكتب clientAuth → localStorage
                                                              ↓
                                                    SiteHeader.tsx (يُظهر اسم المستخدم)
                                                    dashboard.tsx (يقرأ الهوية)
```

## 🛠️ التقنيات المستخدمة

| التقنية | الاستخدام |
|---------|---------|
| React 18 + TypeScript | إطار العمل الرئيسي |
| TanStack Router v1 | نظام المسارات (File-based routing) |
| TanStack Query | إدارة البيانات المتزامنة |
| Vite | أداة البناء والتطوير |
| Tailwind CSS | التنسيق العام للموقع |
| Inline Styles | تنسيق لوحة الأدمن (بدون Tailwind) |
| Recharts | الرسوم البيانية (Overview، Reports) |
| Lucide React | الأيقونات |
| shadcn/ui | مكتبة مكونات الواجهة |
| localStorage | التخزين المؤقت (بديل Backend) |

---

> **ملاحظة مهمة:** كل البيانات تُخزَّن حالياً في `localStorage` — هذا نموذج أولي (Frontend-only). عند دمج Backend، ستُستبدل استدعاءات `store.ts` باستدعاءات API دون الحاجة لتغيير منطق المكونات.
